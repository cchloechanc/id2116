Weekly Progress

Over some weeks, we learned more about some relevant computing methods that eventually helped us to produce our Mini Project 01.

We have documented our learning progress in this section!

Week 01

We learned the basics about micro:bit, its architecture and programming languages.

Some of the concepts taught were:

  • Loops: For iterations and repetitions
  • Variables: Assigning values to a variable for multiple uses
  • Conditionals: Selection of steps to take in a code
  • Sensors: Displays values from external sensors

We were also introduced to the MVC Design Pattern of WWW - Separation of:

  • Model (DB,HTML): Database, document structure
  • View (CSS): Representation, appearance
  • Controller (Javascript): User input, manipulation)

Week 02

We learned about HTML and CSS.

Keywords:

  • HTTP (Hyper Text Transfer Protocol): Communication between client computers and web servers is done by sending HTTP requests and receiving HTTP Responses.
  • URI (Uniform Resource Identifier): A compact sequence of characters that identifies an abstract or physical resource.
  • HTML (Hyper Text Markup Language): The World Wide Web's core markup language. It describes the “structure” of a document.
  • CSS (Cascading Style Sheet): A language that describes the style of an HTML document, describes how HTML elements should be displayed.

Creating Our First GitHub Page:

  • First HTML Code: Learning how to create and use HTML code to set-up our GitHub page. We learned about the main steps “commit”, “push” and “pull”.
  • First CSS Code: We learned how to change color/size/font of text elements created in HTML.
  • Individual GitHub Pages: After learning the skills to create our joined GitHub page, we created our own individual pages!

Week 03

We were introduced to CSS/DOM/Javascript and p5js.

CSS/DOM/Javascript:

  • CSS Selectors - Linking CSS with specific HTML elements: Element Selector, id Selector, Class Selector, Universal Selector
  • CSS Box Model - Placing HTML elements into nested boxes: Margin > Border > Padding

p5js: JavaScript library for "Creative Coding":

  • We started off with an exercise to model a light bulb. We used the property of a light bulb (on: light, off: no light), and linked its brightness to when the mouse is clicked/released.
  • We also completed the challenges of: Toggling the action using a “counter” and setting the on and off function to “odd” and “even” numbers respectively. Fading the light bulb’s brightness using the “tint” and “slider” function.
  • The next exercise of the day was to create a digital photo frame, and ended off the by learning how to load sound and videos onto sketch.

Week 04

We learned about object/OOP/StateTransition and more about micro:bit and p5js.

Object/OOP/StateTransition (Defining our daily object as a Javascript "Object"):

  • Define an object: A container that is capable of taking in named values such as properties and methods. Objects can also have methods - actions that can be performed or function definitions.
  • Object oriented programming: Creates objects that interacts with each other.
  • Class: A type of function to generate an object from a template. A defined class of an object can be called without rewriting the properties.

micro:bit and p5js (Connecting microbit with p5js using bluetooth):

  • We learned to utilise the bluetooth library to communicate with the microbit. Through bluetooth connectivity, we play, pause a video by tilting the microbit.

Mini Project 01: New "Switch"

Hardware: micro:bit | Software: p5js

For this project, we were tasked to transform an “Everyday Object” into a “Digital Switch” to trigger a media experience (light, audio, video and motion).

We have documented our project progress of 7 weeks here!

Week 01

Step 01: Pick an everyday object and analyse its form and function

This week we picked our two everyday objects and analysed them to decide on their function and how we could potentially integrate a new experience to it.

Week 02

Step 02: Embed micro:bit to extract a physical action as digital information

micro:bit hardware control:

  • We learned about the micro:bit hardware control with the target of learning how to eventually add digital I/O to our chosen daily object.
  • We controlled an LED(output) using the “digital write” function on the micro:bit. A push button switch was added as an input to toggle the LED on and off.
  • We also explored with analog read/write function of the micro:bit with the addition of a variable resistor to control the brightness of the LED.
  • P.S. Chloe sucks and broke the adapter attached to the micro:bit and breadboard.

Week 03

Step 03: Development From Week 02

Changing our everyday item into something more interesting.

  • As we were not very interested in the different results that the camera lens and coffee presser could offer, we decided to ideate more and decided to try experimenting with a bubble gun.
  • Based on our analysis (as shown in the table above), we came up with the concept of having an image being the feedback when a single bubble pops. We felt that it was unexpected and interesting feedback: multiple images revealed when multiple bubbles pop as they hit a wall.

Week 04

Step 04: Designing Behaviour of Objects: State Transition

This week we focused on prototyping our concept out to test its feasibility.

  • As we were trying to prototype with a bubble popping to reveal an image, we realised that the contact of a single bubble popping against the wall was not strong enough for the microbit to sense.
  • Hence, we decided to switch our direction to having musical notes being played corresponding to the part of the wall as multiple bubbles pop against it. This will hopefully help the microbit to sense the popping of the bubbles more, as it is likely that when more bubbles pop at the same time, the contact would be stronger.

Week 05

Step 05: Concept Development and Refinement

We decided improve on our narrative and developed more on our concept.

  • We were inspired to strengthen our narrative and to ideate on how the unexpected output could have a stronger correlation with the bubbles coming out from the gun.
  • Eventually, we came up with the concept of having the bubbles act as oxygen supplies, resuscitating, and finally helping the person (in the video) to swim out of the waters, and pop out from the screen (and tank).

Week 06

Step 06: Prototype Refinement and Final Consult

During our recess week, we continued our adventure on refining our prototype and consulted our professor.

  • We started out by testing to see if the bubble can be sensed by microbit when it touches copper tape. However, using this method resulting in a lot of difficulties: The copper tape would often get very wet and soggy. The detection was often very unstable.

  • So we decided to ask for some help, and after consultation, we decided to simplify our prototype, and to change our trigger point to the point where we click the button. (Once the button is clicked, the video will play)
  • So, we modified our plan and this is the final one:
  • The final concept is as such: The visuals will start by showing someone lying at the bottom of the seabed.
  • The user will have to click the button on the bubble gun to supply him with bubbles of oxygen so that he can swim up the sea to escape.
  • However, if the user releases the button, the person in the video will sink back to the bottom of the sea.
  • Once the user has supplied enough oxygen to the person in the video, there will be a final visual to congratulate the user, and a song and confetti popper will pop as a form of celebration.

Our Final Stages in Prototyping:

  • To do this, we had to unscrew and open the camera to attach a sensor to the button, and attach the microbit:
